## Why

Sub-proposal 4.11 of the `test-coverage` initiative — a **HIGH-stakes** capability-flow carve-out, flagged in the parent `tasks.md` as the "privacy-leak class of bug." The parent line scopes the carve-out as "three-state visibility (private | unlisted | public) enforcement in **DAL and server actions**." Spec-grep + a coverage read of `main` at HEAD narrows the *residual* carve-out sharply, because the DAL half of that mandate is already discharged:

- `lib/listAccess.ts` (the `guardListViewable` / `isItemViewable` / `isListViewableForViewer` viewability predicate — the read-path enforcement of the three-state model) and `lib/visibility.ts` (the `fromDb` decoder + `visibilityDbValues` expander) were **already tested to the universal `COVERAGE_FLOOR` by §2.1 `test-pure-libs`** (archived `2026-05-26-test-pure-libs`; tests live at `lib/__tests__/listAccess.test.ts` and `lib/__tests__/visibility.test.ts`, both enumerated in `vitest.config.ts`). This sub-proposal does **not** re-test them — it inherits them.

The residual visibility-enforcement surfaces that no test locks today are the two **mutation / metadata** surfaces:

1. **`setListVisibility`** in `app/actions/lists.ts` — the server action that writes `lists.visibility`. It is the sole code path that performs the `shared_at` transition state machine, the legacy-`shared` dual-write, server-side owner authorization, and fail-closed enum re-validation. Untested.
2. **`generateMetadata`** in `app/(main)/lists/[id]/page.tsx` — the route metadata function whose entire job is to prevent the list *name* from leaking into served `<head>` metadata to non-owners of non-`public` lists, and to emit `noindex` on every list page. This is the privacy-leak surface that makes 4.11 HIGH-stakes. Untested.

Both functions live in **multi-owner files** (the file also contains functions owned by sibling sub-proposals), which the per-file coverage-floor model does not natively express — resolving that tension is the central design problem (see `design.md` D1).

### Inherited constraints (spec-grep across all active specs)

- **`list-visibility`** (active spec, `openspec/specs/list-visibility/spec.md`) — the authoritative, already-comprehensive contract. Six requirements, every one binding on this carve-out: (R1) the three-state access model; (R2) `shared_at` updates only on `private → non-private`, clears on return to private, preserves across `unlisted ↔ public`; (R3) the owner-only three-item radio menu UI; (R4) migration mapping `shared` → `visibility` without retroactive broadcast; (R5) `setListVisibility` dual-writes the legacy `shared` boolean (`shared = visibility != 'private'`); (R6) every list page is `noindex` AND non-`public` lists do not leak names in metadata to non-owners, failing closed. R2/R5 are enforced by `setListVisibility`; R6 by `generateMetadata`. **This carve-out's tests assert against these SHALLs as the authoritative contract** — nearly every invariant is already specced, so this proposal ADDS only one latent SHALL (see Capabilities) and elevates the rest as non-elevations with rationale.
- **`testing-foundation`** (active delta at `openspec/changes/test-coverage/specs/testing-foundation/spec.md`, as amended by §0.1 `test-housekeeping` to a single universal `COVERAGE_FLOOR = lines:98 / statements:98 / branches:95 / functions:100`, plus the archived deltas from `test-foundation*`, `test-pure-libs`, and the primitive-family carve-outs) — every requirement applies verbatim: runner (vitest 4.x, node project for `.test.ts`), four-gate pre-merge, `__tests__/` colocation, single-`COVERAGE_FLOOR` per-file thresholds, no-backdoor disposition rule, `<State>_<Behavior>` `it()` shape, role-grouped `describe()`, the four-audit obligation, the invariant-elevation audit, the assertion-substance bar, `sonarjs/cognitive-complexity` warn-globally / error-per-carve-out, and the NextAuth-as-network-boundary mocking allowance (mock `@/lib/auth`'s `auth()`; never mock internal DAL — exercise it against the real pglite test DB, per `lib/__tests__/listAccess.test.ts`'s established harness).
- **`server-endpoint-authorization`** (active spec; aligns with §4.13 `test-server-endpoint-authorization`, not yet started) — owns the *generic* authorization matrix (owner / authenticated non-owner / unauthenticated) across **every** server action and API route. `setListVisibility`'s authorization checks are therefore **dual-owned**: 4.11 tests them as part of the visibility mutation's complete contract (the `list-visibility` spec's "Non-owner submission is rejected" scenario lives here), and 4.13 will later assert the same action within its cross-action matrix. The overlap is intentional and is called out so 4.13 does not re-litigate it; see `design.md` D5.
- **`following`, `home-digest`** — OWN the follower-feed DAL reads that filter on `visibilityDbValues([VISIBILITY.FOLLOWERS])` (`lib/dal.ts` lines ~598/647/684, consumed by `getFollowing*` and the home digest). Those reads are visibility-*filtered* but are feed behavior, not visibility enforcement per se; they are **out of this carve-out** and belong to §4.2 / §4.3. This carve-out does not touch `lib/dal.ts`.
- **Cross-cutting design-system rule** — the visibility UI (`VisibilityPicker.tsx`, `visibility-rows.tsx`, the trigger pill) is an interactive surface built on the `menu-system` and `popover-trigger-system` primitives (both already tested under §3.4 / §3.5). The `list-visibility` spec's R3 governs that UI. **It is out of this carve-out** (this carve-out is "DAL and server actions," not UI) — the UI's behavior is exercised end-to-end by §6.1 `test-e2e-critical-flows` ("set visibility"). No page-scoped className work is proposed.
- **Data-freshness rule** — `setListVisibility` mutates `lists` and calls `updateTag('lists')`; the test asserts the tag write so a future edit that drops revalidation (staling every `'use cache'` + `cacheTag('lists')` read — `getList`, `getListsByUser`, etc.) fails loudly.

## What Changes

- **NEW** two colocated test files under `__tests__/` directories mirroring the source layout, both running under the **node** vitest project (server-side code, no JSX render), both using the established pglite + `vi.mock('@/db')` + `vi.mock('next/cache')` harness from `lib/__tests__/listAccess.test.ts`, plus `vi.mock('@/lib/auth')` for the session boundary (**post-rebase:** the `setListVisibility` half was merged into the existing `app/actions/__tests__/lists.test.ts` rather than added as a standalone file — see the reconciliation section above):

  - The `setListVisibility` export — **post-rebase:** asserted by the `setListVisibility` `describe` block merged into `app/actions/__tests__/lists.test.ts` (originally planned as a standalone `app/actions/__tests__/lists.setListVisibility.test.ts`). Asserts the full visibility-mutation contract against a real pglite-backed `lists` row:
    - **Authorization (fail-closed, server-resolved actor):** unauthenticated caller (no `auth()` session / no `session.user.email`) → `{ success: false, error: 'Unauthorized' }`, row unchanged; authenticated caller whose email resolves to no `users` row → `{ error: 'Unauthorized' }`, row unchanged; authenticated non-owner → `{ success: false, error: 'Forbidden' }`, row unchanged (locks the `list-visibility` R3 "Non-owner submission is rejected" scenario); non-existent `id` → `{ error: 'Not found' }`.
    - **Fail-closed enum re-validation (the one ADDED SHALL):** a value outside `VISIBILITY_VALUES` (e.g. `'owner'`, `'admin'`, `''`) → `{ error: 'Validation' }` returned **before any DB read of the target list or any UPDATE** — asserted by confirming the row's `visibility`, `shared`, and `shared_at` are byte-for-byte unchanged. This locks the invariant that the server action re-validates a parameter the TypeScript type (`ListVisibility`) claims is already safe, because the type is erased at the server-action network boundary.
    - **`shared_at` transition state machine (R2):** `private → unlisted` and `private → public` set `shared_at = NOW()` (assert it moves from `null` to a fresh timestamp ≈ test clock); `unlisted → public` and `public → unlisted` **preserve** the prior `shared_at` (assert byte-identical to the pre-call value); `public → private` and `unlisted → private` set `shared_at = null`; the full cycle `public → private → public` yields a **second** `shared_at` strictly newer than the first (assert `> first`).
    - **Legacy `shared` dual-write (R5):** after `→ public` and after `→ unlisted`, the row's `shared = true`; after `→ private`, `shared = false`. Asserted by reading the row back, since dev code never reads `shared`.
    - **Cache revalidation:** `updateTag('lists')` is called exactly once on the success path (spied via the `next/cache` mock); it is NOT called on any early-return failure path (unauthorized / validation / not-found / forbidden).
    - **Success shape:** `{ success: true, message: 'Visibility updated' }`.
    - **No-op self-transition:** `private → private` (and `public → public`) leaves `shared_at` and `shared` consistent (no spurious `shared_at` set on `private → private`; `shared_at` preserved/identity on `public → public`) — locks the `wasPrivate`/`goingPrivate` branch interaction.

  - `app/(main)/lists/[id]/__tests__/page.generateMetadata.test.ts` — exercises **only** the `generateMetadata` export of `page.tsx` (the default `ListPage` component export is owned by the list-page render carve-out, not this one). Runs in the node project; `getList` / `getUserIdByEmail` are exercised against the real pglite DB (per testing-foundation: do NOT mock internal DAL), and `@/lib/auth`'s `auth()` is mocked at the session boundary. Asserts the full leak-prevention contract (R6):
    - **`noindex` is universal:** for `private`, `unlisted`, AND `public` lists, and for the not-found / `getList`-throws fallback, the returned `Metadata.robots` deep-equals `{ index: false, follow: false }`.
    - **Leak prevention for non-owner of non-`public`:** non-owner (authenticated *or* anonymous) of a `private` list → `title === 'List | ctrl+list'` (the `GENERIC_LIST_TITLE` constant), `openGraph` and `twitter` blocks **absent** (assert `undefined`); same for `unlisted`.
    - **Full metadata for owner of own non-`public`:** owner of their `private` / `unlisted` list → `title === list.name`, `openGraph.title === list.name`, `twitter` block present, AND `robots` still `noindex`.
    - **Full metadata for `public` to everyone:** `public` list → full `title` / `openGraph` / `twitter` for owner, authenticated non-owner, AND anonymous viewer, with `auth()` **not consulted** on the `public` path (assert the `auth` mock recorded zero calls — locks the `isShared` short-circuit that skips the session lookup).
    - **Fail-closed fallbacks:** `getList` resolves `null` (unknown id) → generic title + `noindex`, no OG/twitter; `getList` throws → same. Asserts the `try/catch` and the `if (!list)` guard both fail closed.

- **NO** test for `lib/listAccess.ts`, `lib/visibility.ts` — already covered by §2.1 (archived). Inherited, not re-tested.
