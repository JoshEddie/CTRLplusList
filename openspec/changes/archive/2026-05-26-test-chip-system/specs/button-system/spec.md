## REMOVED Requirements

### Requirement: Removable chips share the button focus/touch contract via `<Chip>` primitive

**Reason**: The chip primitive is a sibling component to `<Button>` / `<LinkButton>` (it lives at `app/ui/components/chip/`, not `app/ui/components/button/`), and its requirement-set has grown beyond the focus/touch contract this single SHALL captured. Continuing to govern it from `button-system` muddies the boundary: a future reader looking for chip behavior would not expect to find it in the button spec. Per the parent `test-coverage` change's §3.2 elevation invitation, the chip primitive is moved into its own dedicated capability spec.

**Migration**: The verbatim content of this requirement (description + all four scenarios — "Active-filter chip renders in the items toolbar", "Selected-list chip renders in the item-form list picker", "Chip's focus and touch contract matches buttons", "Legacy chip class definitions are removed") is preserved as the first requirement in the new `chip-system` capability spec (`openspec/specs/chip-system/spec.md`), with no behavior change. Additional chip-call-time SHALLs (DOM shape, class composition, `aria-label` derivation, `type="button"` + `stopPropagation` semantics, `disabled` passthrough) are elevated alongside it in the new spec. Call sites are unchanged — `<Chip>` continues to be imported from `app/ui/components/chip/` and consumed exactly as before. The `button-system` requirement "Small-size button opt-out with documented spacing" (which references chips as a valid `<Button size="sm">` context) is NOT moved — it governs button sizing, not the chip primitive, and stays in `button-system`.
