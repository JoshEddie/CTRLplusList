- **8.1 Assertion-substance — PASS.** Every `it()` asserts an observable: a redirect sentinel (`rejects.toThrow('REDIRECT:/…')` in the RSC shells), a forwarded prop on a prop-surfacing stub (`data-*` attributes), a mocked-action call payload (`createPurchase({ item_id, guest_name })`, `updatePriority(id, target, listId)`, `setListItems(list_id, [...])`, `archiveItem(id, bool)`), a `router` navigation arg, or rendered DOM/text. One bad assertion was found and fixed in place during authoring (`QuantityLimitField` `'abc'`-input test asserted a non-event); removed.
- **8.2 Duplication — keep-separate (named reason).** The `auth()`-mock + `redirect`-sentinel (`vi.hoisted` throwing `REDIRECT:<url>`) + per-file `@/db` / `@/lib/dal` mocks recur across the four RSC-shell test files (`ChooseItemsBody`, `ItemFormBody`, `SortItemsContainer`, `ListItemsSection`) and the two route shells. Per the established precedent recorded in `app/actions/__tests__/test-helpers.ts` ("the vi.mock harness stays inline per file because vi.mock is hoisted per module"), these are NOT extracted — `vi.mock`/`vi.hoisted` factories cannot be shared from a helper module. Only the pure fixture builders are reused (`makeItem`/`renderItem` etc. are local per file; the action-layer `seed*` builders were not needed since the shells mock the DAL boundary per Decision 4). `returnTo` validation is not duplicated — it already lives in the single shared `returnTo.ts` (`sanitizeReturnTo`), exercised through `ItemFormBody.test.tsx` (valid `/…`, `//`, `://`, `\`, no-leading-slash, fallback `/items`). No new `utils.ts`/`test-helpers.ts` extracted.
